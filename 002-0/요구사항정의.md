## 1. 프로젝트 개요

### 1.1 목적
YouTube 영상의 자막을 자동으로 추출, 번역, 요약하여 구조화된 정보를 제공하고 Notion에 저장하는 시스템 개발

### 1.2 목표
- YouTube 영상 URL 입력만으로 전체 콘텐츠 요약 자동화
- 자막이 없는 영상도 Whisper API를 통해 처리
- 타임라인별 상세 내용과 주제별 섹션 분류 제공
- Notion 연동으로 학습/업무 자료 자동 정리

### 1.3 범위
- **포함**: YouTube 자막 추출, AI 기반 번역/요약, Notion API 연동, REST API 제공
- **제외**: 유료 영상 처리, 실시간 스트리밍 영상, 비공개 영상

#### 상세 처리 플로우

**1. 영상 정보 조회 플로우**
```
사용자 요청 (YouTube URL)
    ↓
[1] URL 검증
    ↓
[2] Video ID 추출
    ↓
[3] yt-dlp로 영상 정보 요청
    ├─ 성공 → 영상 정보 반환
    └─ 실패 (400 에러)
        ↓
    [4] 쿠키 기반 인증 재시도
        ├─ Chrome 쿠키 로드
        ├─ Android 클라이언트 에뮬레이션
        └─ 재요청
            ├─ 성공 → 영상 정보 반환
            └─ 실패 → 에러 반환
```

**2. 자막 추출 플로우**
```
Video ID 입력
    ↓
[1] YouTube Transcript API 호출
    ↓
[2] 자막 우선순위 검색
    ├─ 한국어 자막 존재? → 추출
    ├─ 영어 자막 존재? → 추출
    ├─ 자동 생성 자막 존재? → 추출
    └─ 자막 없음
    ↓
[3] Whisper API 사용 안내
    ├─ 오디오 다운로드
    ├─ Whisper API 호출
    └─ 텍스트 변환
    ↓
[4] 타임라인 형식 변환
    ├─ 타임스탬프 포맷팅 (00:00)
    ├─ 텍스트 정제
    └─ 구조화된 데이터 생성
    ↓
타임라인 데이터 반환
```

## 2. 기능 요구사항

### 2.1 핵심 기능

#### FR-001: YouTube 영상 정보 추출
- **우선순위**: 높음
- **설명**: YouTube URL로부터 영상의 기본 정보(제목, 채널명, 재생시간) 추출
- **입력**: YouTube URL (예: https://www.youtube.com/watch?v=VIDEO_ID)
- **출력**:
  - video_id: 영상 ID
  - title: 영상 제목
  - author: 채널명
  - length: 재생 시간(초)
- **검증 기준**:
  - 유효한 YouTube URL 검증
  - 공개 영상만 처리
  - 400/403 에러 발생 시 쿠키 기반 인증으로 재시도

#### FR-002: 자막 추출
- **우선순위**: 높음
- **설명**: YouTube 영상에서 자막 데이터 추출
- **입력**: YouTube video_id
- **출력**: 타임라인별 자막 데이터
  - time: 타임스탬프 (00:00 형식)
  - text: 자막 텍스트
  - start: 시작 시간(초)
  - duration: 지속 시간(초)
- **우선순위**:
  1. 한국어 자막
  2. 영어 자막
  3. 자동 생성 자막
  4. 자막 없음 → Whisper API 처리
- **예외 처리**:
  - 자막 비활성화 시: 에러 메시지 반환
  - 자막 없음 시: Whisper API 사용 안내

#### FR-003: 자막 번역 및 정제
- **우선순위**: 높음
- **설명**: 추출한 자막을 한국어로 번역하고 문맥에 맞게 정제
- **입력**: 원본 자막 데이터
- **처리**:
  - OpenAI GPT-4o-mini 사용
  - 청크 단위 번역 (긴 영상 대응)
  - 문맥 보존 및 자연스러운 표현 변환
- **출력**: 한국어로 번역된 타임라인 데이터

#### FR-004: 섹션 분류 및 요약
- **우선순위**: 높음
- **설명**: 영상 내용을 주제별 섹션으로 분류하고 각 섹션 요약
- **입력**: 번역된 타임라인 데이터
- **처리**:
  - 청크 기반 분석 (100개 항목씩)
  - AI 기반 주제 추출 및 병합
  - 계층화 단계 (10개 이상 주제 시)
    - 대주제-소주제 구조화
    - 유사 주제 그룹핑
    - 80% 이상 데이터 보존 검증
- **출력** (계층 구조):
  - overview: 영상 개요
  - sections[]:
    - section_title: 대주제 제목
    - section_summary: 섹션 요약
    - subtopics[]:
      - title: 소주제 제목
      - content: 상세 내용
  - key_insights: 핵심 인사이트
  - key_terms[]: 핵심 용어 목록

#### FR-005: 전체 요약 생성
- **우선순위**: 중간
- **설명**: 영상 전체 내용을 한눈에 파악할 수 있는 요약문 생성
- **입력**: 섹션별 요약 데이터
- **출력**: 2-3 문단의 전체 요약문

#### FR-006: Notion 페이지 생성
- **우선순위**: 중간
- **설명**: 처리된 결과를 Notion 페이지로 자동 생성
- **입력**:
  - 영상 정보
  - 타임라인
  - 섹션 요약
- **출력**: Notion 페이지 URL
- **페이지 구조**:
  - 영상 제목 (헤딩1)
  - 기본 정보 (채널, 재생시간, URL)
  - 전체 요약
  - 섹션별 요약 (접기/펼치기)
  - 타임라인 전체 내용

#### FR-007: 빠른 처리 모드
- **우선순위**: 낮음
- **설명**: 테스트 및 미리보기용 빠른 처리 기능
- **입력**: YouTube URL, 제한 개수 (기본: 50)
- **처리**: 처음 N개 자막 항목만 처리
- **출력**: 제한된 데이터로 생성된 요약

### 2.2 API 엔드포인트

#### API-001: POST /api/video-info
- **기능**: 영상 정보 조회
- **요청**:
  ```json
  {
    "url": "https://www.youtube.com/watch?v=VIDEO_ID"
  }
  ```
- **응답**:
  ```json
  {
    "video_id": "VIDEO_ID",
    "title": "영상 제목",
    "author": "채널명",
    "length": 212,
    "url": "https://www.youtube.com/watch?v=VIDEO_ID"
  }
  ```

#### API-002: POST /api/extract-transcript
- **기능**: 자막 추출
- **요청**: 동일 (video-info)
- **응답**:
  ```json
  {
    "success": true,
    "video_info": {...},
    "timeline": [...],
    "language": "ko",
    "source": "youtube",
    "total_items": 100
  }
  ```

#### API-003: POST /api/process-video
- **기능**: 전체 처리 파이프라인
- **요청**: 동일 (video-info)
- **응답**:
  ```json
  {
    "success": true,
    "video_info": {...},
    "timeline": [...],
    "sections": [...],
    "full_summary": "...",
    "notion_url": null,
    "language": "ko",
    "source": "youtube",
    "message": "영상 처리 완료!"
  }
  ```

#### API-004: POST /api/process-video-quick
- **기능**: 빠른 처리 (테스트용)
- **요청**: 동일 (video-info)
- **응답**: 동일 (process-video), 단 제한된 데이터

## 3. 비기능 요구사항

### 3.1 성능 요구사항
- **NFR-001**: 영상 정보 조회는 5초 이내 완료
- **NFR-002**: 10분 영상 전체 처리는 3분 이내 완료
- **NFR-003**: 1시간 영상 전체 처리는 10분 이내 완료
- **NFR-004**: API 응답 시간 95%는 2초 이내

### 3.2 안정성 요구사항
- **NFR-005**: YouTube API 실패 시 재시도 메커니즘 (최대 3회)
- **NFR-006**: OpenAI API 실패 시 명확한 에러 메시지 반환
- **NFR-007**: 서버 크래시 없이 모든 에러 핸들링

## 10. 용어 정의

- **타임라인**: 영상의 시간순 자막 데이터
- **섹션**: 주제별로 분류된 영상 구간
- **Agent**: LangChain 기반 AI 처리 파이프라인
- **Chain**: 특정 작업을 수행하는 AI 모듈
- **Whisper**: OpenAI의 음성→텍스트 변환 API
- **yt-dlp**: YouTube 다운로더 라이브러리
- **청크**: 긴 데이터를 처리 가능한 크기로 분할한 단위
- **계층화**: 평탄한 주제 목록을 대주제-소주제 구조로 재구성
- **Hierarchize Chain**: 주제 목록을 계층 구조로 변환하는 AI 모듈
