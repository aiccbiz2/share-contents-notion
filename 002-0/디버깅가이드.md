## 1. 로그 시스템

### 1.1 로그 파일 위치
```
backend/logs/
├── app_YYYYMMDD.log      # 일반 로그 (INFO 이상)
├── error_YYYYMMDD.log    # 에러 로그 (ERROR 이상)
└── ...
```

### 1.2 로그 레벨
- **DEBUG**: 상세한 디버깅 정보
- **INFO**: 일반 정보 (API 요청, 처리 상태)
- **WARNING**: 경고 메시지 (재시도, 대체 방법 사용)
- **ERROR**: 에러 발생 (스택 트레이스 포함)

### 1.3 로그 파일 특징
- 파일 크기 제한: 10MB
- 백업 파일 수: 5개
- 인코딩: UTF-8
- 자동 로테이션: 파일 크기 초과 시

## 3. 일반적인 문제 해결

### 3.1 YouTube 400 에러

**증상:**
```
HTTP Error 400: Bad Request
```

**원인:**
- YouTube의 봇 차단
- 쿠키 인증 실패

**해결방법:**
1. Chrome 브라우저 설치 확인
2. Chrome에 YouTube 로그인 상태 확인
3. 서버 재시작
4. 로그 확인:
   ```bash
   GET http://localhost:8000/debug/logs?lines=50&log_type=error
   ```

### 3.2 OpenAI API 에러

**증상:**
```json
{
  "success": false,
  "error": "Invalid API key"
}
```

**해결방법:**
1. `.env` 파일 확인:
   ```bash
   OPENAI_API_KEY=sk-proj-...
   ```
2. API 키 테스트:
   ```bash
   POST http://localhost:8000/debug/test-openai
   ```
3. OpenAI 대시보드에서 API 키 유효성 확인

### 3.3 로그 파일이 생성되지 않음

**확인 사항:**
1. `backend/logs/` 디렉토리 존재 여부
2. 디렉토리 쓰기 권한
3. 디스크 공간 여부

**해결방법:**
```bash
# 로그 디렉토리 수동 생성
mkdir backend/logs

# 권한 확인 (Linux/Mac)
ls -la backend/logs
```

### 3.4 서버가 시작되지 않음

**확인 순서:**
1. 가상환경 활성화 확인
2. 패키지 설치 확인:
   ```bash
   pip list | grep fastapi
   ```
3. 포트 충돌 확인:
   ```bash
   # Windows
   netstat -ano | findstr :8000

   # Mac/Linux
   lsof -i :8000
   ```
4. 로그 확인

## 4. 디버깅 워크플로우

### 4.1 일반적인 디버깅 절차

```
1. 문제 발생
   ↓
2. 에러 메시지 확인 (API 응답)
   ↓
3. 로그 조회
   GET /debug/logs?log_type=error
   ↓
4. 시스템 상태 확인
   GET /debug/system-info
   GET /debug/health-detailed
   ↓
5. 구성 요소별 테스트
   POST /debug/test-youtube
   POST /debug/test-openai
   ↓
6. 문제 원인 파악 및 해결
   ↓
7. 재테스트
```

### 4.2 YouTube 처리 디버깅

```bash
# 1. YouTube 연결 테스트
POST /debug/test-youtube
{
  "url": "YOUR_YOUTUBE_URL"
}

# 2. 영상 정보만 가져오기
POST /api/video-info
{
  "url": "YOUR_YOUTUBE_URL"
}

# 3. 자막 추출만 테스트
POST /api/extract-transcript
{
  "url": "YOUR_YOUTUBE_URL"
}

# 4. 로그 확인
GET /debug/logs?lines=100&log_type=app
```

### 4.3 AI 처리 디버깅

```bash
# 1. OpenAI 연결 테스트
POST /debug/test-openai

# 2. 빠른 처리로 테스트 (50개 항목만)
POST /api/process-video-quick
{
  "url": "YOUR_YOUTUBE_URL"
}

# 3. 로그 확인
GET /debug/logs?lines=200&log_type=app
```

## 6. 성능 모니터링

### 6.1 실시간 리소스 확인
```bash
GET /debug/health-detailed
```

주의사항:
- CPU 사용률 80% 이상: 동시 요청 과다
- 메모리 사용률 90% 이상: 메모리 부족, 서버 재시작 필요

### 6.2 처리 시간 측정

로그에서 타임스탬프 차이로 계산:
```
시작: 2025-11-27 09:20:15

```
