# 📰 Discord 뉴스 URL → Notion 자동 분석 시스템 개발 계획서

> **프로젝트명:** Discord News Notion Bot  
> **작성일:** 2026-02-21  
> **버전:** v1.1  
> **목표:** 유저가 디스코드에 뉴스 URL을 붙여넣으면, 봇이 자동으로 AI 분석 후 Notion에 저장

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적
뉴스를 읽다가 인상깊은 기사를 디스코드 채널에 URL로 공유하면, AI가 자동으로 분석하여 Notion에 구조화된 형태로 저장한다. 매번 직접 요약을 적는 수고 없이 지식 아카이브를 자동으로 쌓아가는 것이 목적이다.

### 1.2 핵심 워크플로우

```
👤 유저가 기사를 읽다가 URL 복사
    ↓
💬 디스코드 특정 채널에 URL 붙여넣기  ← 🔥 트리거 포인트
    ↓
🤖 Discord Bot이 메시지에서 URL 감지
    ↓
🌐 URL에서 기사 본문 가져오기 (on-demand fetch)
    ↓
🧠 Gemini AI로 3가지 관점 분석
    ↓
📓 Notion DB에 분석 결과 페이지 생성
    ↓
💬 디스코드에 완료 메시지 + Notion 링크 회신
```

> ⚠️ **중요:** 봇은 뉴스 사이트를 자동으로 모니터링하거나 크롤링하지 않습니다.  
> 오직 **유저가 디스코드에 URL을 공유했을 때만** 동작합니다.

### 1.3 핵심 기능
| 기능 | 설명 |
|------|------|
| URL 감지 | 디스코드 메시지에서 URL 자동 감지 |
| 기사 본문 추출 | 해당 URL에서 기사 제목·본문 on-demand 추출 |
| AI 분석 | 핵심요약 / Implication / 비판적 사고 관점 생성 |
| Notion 저장 | 분석 결과를 Notion DB 페이지로 자동 생성 |
| 완료 알림 | 디스코드에 리액션 + Notion 페이지 링크 회신 |

## 4. 상세 모듈 설계

### 4.1 `bot/events.py` - 트리거 핸들러 ⭐ 핵심

**역할:** 유저가 URL을 디스코드에 붙여넣으면 파이프라인 시작

```python
# 처리 흐름 (의사코드)
@bot.event
async def on_message(message):
    if message.author.bot: return          # 봇 자신의 메시지 무시
    if not is_target_channel(message): return  # 지정 채널 외 무시

    urls = extract_urls(message.content)   # 메시지에서 URL 추출
    if not urls: return                    # URL 없으면 무시

    await message.add_reaction("⏳")       # 처리 중 표시

    for url in urls:
        try:
            article = await fetcher.fetch(url)       # 기사 본문 추출
            analysis = await analyzer.analyze(article)  # AI 분석
            notion_page = await notion.save(analysis, message.author.name)  # Notion 저장
            
            await message.reply(f"✅ 분석 완료!\n{notion_page.url}")
        except Exception as e:
            await message.reply(f"❌ 처리 실패: {str(e)}")

    await message.add_reaction("✅")
```

**채널 설정:**
- `.env`의 `DISCORD_CHANNEL_IDS`에 지정한 채널만 감지
- 비워두면 서버 전체 채널 감지

### 4.3 `services/analyzer.py` - AI 분석

**역할:** 추출된 기사를 Gemini API로 분석, 3가지 관점 생성

**입력:** `ArticleData`  
**출력:**
```python
@dataclass
class AnalysisResult:
    one_line_summary: str   # 한 줄 요약 (디스코드 회신용)
    key_summary: list[str]  # 핵심요약 불릿 포인트 3~5개
    implication: str        # 시사점 (단기/중장기 영향)
    critical_thinking: str  # 비판적 사고 관점 + 독자 질문
    tags: list[str]         # 자동 카테고리 태그
```

**Gemini 프롬프트 핵심:**
```
## 출력 형식 (JSON)
{
  "one_line_summary": "60자 이내 한 줄 요약",
  "key_summary": ["포인트1", "포인트2", "포인트3"],
  "implication": "단기/중장기 영향, 이해관계자 관점 (300자)",
  "critical_thinking": "빠진 관점, 반대 의견, 독자 질문 (300자)",
  "tags": ["AI", "경제", "테크"]
}
```

### 4.4 `services/notion_service.py` - Notion 저장

**역할:** 분석 결과를 Notion DB에 페이지로 저장

**Notion DB 스키마:**
| 속성명 | 타입 | 설명 |
|--------|------|------|
| 제목 | Title | 기사 제목 |
| URL | URL | 원문 링크 |
| 언론사 | Select | 발행 매체 |
| 발행일 | Date | 기사 발행 날짜 |
| 공유자 | Rich text | 디스코드에 공유한 사람 |
| 카테고리 | Multi-select | AI 자동 생성 태그 |
| 저장일 | Date | Notion 저장 날짜 (자동) |

**Notion 페이지 본문 구조:**
```
📌 핵심요약          [Callout 블록]
  • 요약 포인트 1
  • 요약 포인트 2
  • 요약 포인트 3

💡 Implication       [Callout 블록]
  시사점 내용 (단기 / 중장기 영향)

🔍 비판적 사고 관점  [Callout 블록]
  비판적 분석 + 독자 질문

─────────────────────
🔗 원문              [원문 URL]
```

## 8. 셋업 가이드

### Discord 봇 설정 (5분)
1. [Discord Developer Portal](https://discord.com/developers/applications) → New Application
2. Bot 탭 → 봇 생성 → Token 복사
3. **Privileged Gateway Intents** 활성화:
   - `MESSAGE CONTENT INTENT` ✅ (필수! URL 읽기에 필요)
4. OAuth2 → URL Generator → `bot` 스코프
   - 권한: `Send Messages`, `Add Reactions`, `Read Message History`
5. 서버에 봇 초대

### Notion 설정 (5분)
1. Notion → Settings → Integrations → New Integration 생성
2. 새 데이터베이스 생성 → **DB 우측 상단 ... → Add connections → 위 Integration 연결**
3. DB URL에서 Database ID 복사 → `.env` 설정
