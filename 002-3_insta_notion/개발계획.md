# 002-3 Instagram → Notion Pipeline (Claude CLI 기반)

## Context

디스코드에 매일 수동으로 수집하는 인스타그램 URL(1~10개/일)의 콘텐츠를 텍스트로 변환해서 기존 노션 지식베이스 DB에 저장하고 싶다. 인스타그램 포스트 유형이 다양(단일 이미지, 캐러셀, 릴스/영상, 스토리 등)하고, 이미지 속 텍스트(OCR), 캡션, 영상 음성 등 모든 정보를 추출해야 한다.

**002-1, 002-2와 동일한 Claude CLI 기반 아키텍처를 사용한다.**

---

## 아키텍처 (002-1/002-2 패턴 동일)

```
Discord 채널 (수동 URL 복붙)
       ↓
discord_bot.py — Instagram URL 감지 → SQLite 큐 저장
       ↓
process_queue.py (launchd 30분 주기 실행)
  ├─ 큐에서 pending 항목 꺼냄
  ├─ extract_instagram.py — instagrapi로 메타데이터 + 미디어 다운로드
  ├─ convert_content.py — 포스트 유형별 텍스트 변환
  │    ├─ 이미지 → Claude Vision API (OCR + 요약)
  │    ├─ 캐러셀 → 각 이미지 개별 Vision 처리 후 통합
  │    ├─ 릴스/영상 → ffmpeg 오디오 추출 → Whisper 음성→텍스트
  │    └─ 캡션/해시태그 → 그대로 보존
  └─ claude -p --dangerously-skip-permissions "{프롬프트}"
       ↓
Claude CLI (CLAUDE.md 읽고 Notion MCP로 직접 저장)
       ↓
Discord 완료 알림 (노션 링크 포함)
```

## 오픈소스 도구 검토 결과

### 추천: `instagrapi` (Python)

| 항목 | 내용 |
|------|------|
| 언어 | Python 3.9+ |
| 유지보수 | 활발 (2025년 v2.2.1) |
| 지원 포스트 | 단일이미지, 캐러셀, 릴스, 영상, 스토리, IGTV |
| 추출 가능 | 캡션, 해시태그, 댓글, 미디어URL, 위치, 사용자정보 |
| 인증 | 필요 (username/password 또는 sessionid) |
| 장점 | URL shortcode로 포스트 조회 가능, 자동 challenge resolver |

### 비추천

| 도구 | 이유 |
|------|------|
| instagram-private-api (Node) | 2024년부터 미유지보수, 보안 취약점 |
| instafeed.js | 프론트엔드 디스플레이용, API 폐기됨 |
| Instagram Basic Display API | 2024.12 공식 폐기 |
| Instaloader | 벌크 다운로드 특화, 실시간 파이프라인 통합에 불편 |

### 주의사항
- instagrapi는 비공식 Private API 사용 → Instagram 약관 위반 가능성
- throwaway 계정 사용 권장
- 일 1~10개 수준이면 차단 리스크 매우 낮음

### 4. `extract_instagram.py` (신규 - 핵심 모듈)

```python
def extract(shortcode, config):
    """
    instagrapi로 포스트 메타데이터 + 미디어 다운로드

    Returns:
        {
            "success": bool,
            "post_type": "image" | "carousel" | "reel" | "video",
            "caption": str,
            "hashtags": [str],
            "username": str,
            "posted_at": str,
            "media_files": [str],   # 다운로드된 파일 경로들
            "media_count": int,
            "location": str | None,
            "error": str | None,

    """
```

- instagrapi Client 로그인 (세션 재사용)
- `media_pk_from_code(shortcode)` → `media_info(media_pk)` 호출
- 포스트 유형 판별: `media_type` (1=Photo, 2=Video, 8=Album)
- 미디어 다운로드: `photo_download()`, `video_download()`, `album_download()`
- 메타데이터 추출: caption, hashtags, location, user info

### 5. `convert_content.py` (신규 - 미디어→텍스트 변환)

```python
def convert(extract_result, config):
    """
    추출된 미디어를 텍스트로 변환

    포스트 유형별 처리:
    - 이미지: Claude Vision API로 OCR + 설명
    - 캐러셀: 각 이미지 개별 Vision 처리 후 통합
    - 릴스/영상: ffmpeg 오디오 추출 → Whisper 변환

    Returns:
        {
            "success": bool,
            "extracted_text": str,      # 이미지/영상에서 추출한 텍스트
            "caption": str,             # 원본 캡션
            "hashtags": [str],
            "post_type": str,
            "media_count": int,
            "summary": str,             # (선택) AI 요약
        }
    """
```

**이미지 → 텍스트 (Claude Vision API)**
```python
import anthropic
import base64

def extract_text_from_image(image_path):
    client = anthropic.Anthropic()
    with open(image_path, "rb") as f:
        image_data = base64.standard_b64encode(f.read()).decode("utf-8")

    message = client.messages.create(
        model="claude-sonnet-4-5-20250929",
        max_tokens=2000,
        messages=[{
            "role": "user",
            "content": [
                {"type": "image", "source": {"type": "base64", "media_type": "image/jpeg", "data": image_data}},
                {"type": "text", "text": "이 인스타그램 이미지의 모든 텍스트를 추출하고, 내용을 요약해줘. 한국어로 답변해줘."}
            ]
        }]
    )
    return message.content[0].text
```

**영상 → 텍스트 (Whisper)**
```python
import whisper
import subprocess

def extract_text_from_video(video_path):
    # 오디오 추출
    audio_path = video_path.rsplit(".", 1)[0] + ".mp3"
    subprocess.run(["ffmpeg", "-i", video_path, "-q:a", "0", "-map", "a", audio_path],
                   capture_output=True)

    # Whisper 변환
    model = whisper.load_model("small")
    result = model.transcribe(audio_path, language="ko")
    return result["text"]
```

### 6. `process_queue.py` (002-1 패턴 + Instagram 추출 단계 추가)

처리 흐름:
```
Step 1: extract_instagram.extract(shortcode) → 메타데이터 + 미디어
Step 2: convert_content.convert(extract_result) → 텍스트 변환
Step 3: claude -p --dangerously-skip-permissions "{프롬프트}" → 노션 저장
```

Claude CLI 프롬프트 템플릿:
```
다음 인스타그램 포스트를 정리해서 노션에 저장해줘.

포스트 유형: {post_type}
작성자: @{username}
게시일: {posted_at}
원본 URL: {instagram_url}

[캡션]
{caption}

[해시태그]
{hashtags}

[이미지/영상에서 추출한 텍스트]
{extracted_text}
```

| 포스트 유형 | URL 패턴 | media_type | 추출 전략 |

| 단일 이미지 | `/p/{code}/` | 1 (Photo) | Vision OCR + 캡션 |
| 캐러셀 | `/p/{code}/` | 8 (Album) | 각 이미지 Vision → 통합 + 캡션 |
| 릴스 | `/reel/{code}/` | 2 (Video) | Whisper 음성변환 + 캡션 |
| 일반 영상 | `/p/{code}/` | 2 (Video) | Whisper 음성변환 + 캡션 |
| 스토리 | `/stories/{user}/{id}/` | 1 or 2 | (24시간 소멸 주의) Vision 또는 Whisper |
