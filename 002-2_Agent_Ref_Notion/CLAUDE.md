# YouTube → Notion 콘텐츠 처리 Instruction

## 기본 규칙
- 영어 콘텐츠는 한글 번역 후 정제
- 한글 콘텐츠는 그대로 정제하여 일목요연하게 정리
- 1000줄 이내 (노션 저장 호환), 내용 적으면 채울 필요 없음
- rating 속성은 건드리지 않음 (사용자가 직접 설정)
- 도구 승인은 항상 YES, 묻지 말 것

###조건:
- 문단 중심으로 작성하되, 문단이 길어지면 핵심 포인트를 bullet으로 분리하여 가독성을 확보할 것
- 정보 나열이 아니라 '이야기 흐름'이 느껴지게 구성할 것
- 소설이나 칼럼을 읽는 것처럼 자연스럽고 가독성 좋게 쓸 것
- "무슨 일이 있었는지 → 왜 중요한지 → 그래서 어떤 변화가 생기는지"
  이 흐름을 유지할 것
- 독자가 중간에 흥미를 잃지 않도록, 맥락과 예시를 적절히 섞을 것
- 과장된 표현이나 유튜브식 말투는 피하고, 차분한 해설 톤으로 작성할 것

###포맷팅 규칙:
- 섹션 제목(heading)과 본문 사이에 빈 줄을 넣어 시각적 여백을 줄 것
- 하나의 문단은 3~5줄을 넘기지 말 것. 길어지면 나눌 것
- 구체적 사례, 수치, 기능 나열 등은 bullet list로 정리할 것
- 문단 → bullet → 문단 흐름을 섞어서 읽는 리듬을 만들 것
- 긴 텍스트 덩어리가 연속으로 붙지 않게 할 것
- 핵심 인사이트나 주목할 수치는 callout 블록으로 강조할 것
- 인물의 발언, 공식 입장 등 직접 인용은 quote(인용) 블록을 활용할 것
- callout과 quote를 적절히 섞어 시각적 변화를 주되, 남용하지 말 것 (섹션당 1~2개)

###출력 형식:
1. 한 문단짜리 도입부 (왜 이 이야기를 읽어야 하는지)
2. 본문 (스토리처럼 이어지는 여러 문단 + 핵심은 bullet으로 시각 분리)
3. 마지막에 짧은 정리 문단 (이 변화가 의미하는 것)

###목표:
노션에 정리했을 때
'요약본'이 아니라 '읽을 가치가 있는 글'처럼 느껴지게 할 것.


## 작업 순서
1. Notion "Agent References" (DB ID: `28ab5922-0286-804c-8b6f-e9950825fb86`) 에 페이지 생성 (제목만)
2. **YouTube URL이 있으면 페이지 생성 직후, 본문 콘텐츠를 추가하기 전에 아래 작업을 먼저 수행할 것:**
   - 첫 번째 블록: type="image"인 블록으로 `https://img.youtube.com/vi/{VIDEO_ID}/maxresdefault.jpg` 이미지를 추가 (embed나 bookmark이 아님! 반드시 image 블록)
   - 두 번째 블록: YouTube 원본 URL 텍스트 링크
   - 이 두 블록이 추가된 후에 본문 콘텐츠를 이어서 작성할 것
   - **절대 cover에만 넣고 본문에 이미지를 빠뜨리지 말 것**

3. 정제된 콘텐츠를 섹션별로 업로드 (실패 시 해당 섹션만 재시도)
4. Cover/Icon 설정 → 이모티콘은 모두 제외할 것

## Cover/Icon 우선순위

| 우선순위 | 조건 | 처리 |
|---------|------|------|
| 1 | YouTube 비디오 있음 | 썸네일 → Cover |
| 2 | 이미지 블록 있음 | 첫 번째 이미지 → Cover |
| 3 | 링크/북마크만 있음 | OG 이미지 |
| 4 | 콘텐츠 없음 | 제목 기반 |

## 주의사항
- Cover는 Notion MCP 도구로 직접 설정할 것 (API 키 없다는 핑계 금지)
- 콘텐츠가 길면 섹션별 분할 업로드, 전체 재작업 금지
- 긴 콘텐츠를 한 번에 전송 시도 금지
- 승인 여부 물어보기 금지
